## ⚖️ Legal & Ethical Testing Guidelines

When working with demo applications and practice environments, I follow responsible software testing principles.

### ✅ Test Only Authorized Systems

Only perform testing on:

- Demo applications
- Sandbox environments
- Personal projects
- Training environments
- Systems where explicit permission has been granted

Never perform testing against production systems without authorization.

---

### ✅ Respect Terms of Service

Before testing any application:

- Review the website’s Terms of Service
- Understand acceptable use policies
- Follow any published testing guidelines
- Respect platform restrictions

---

### ✅ Protect User Privacy

Responsible testing includes protecting personal information.

Avoid:

- Accessing confidential information
- Collecting personal user data
- Sharing sensitive information
- Attempting to bypass privacy controls

---

### ✅ Practice Responsible Security Testing

Security learning should always remain ethical.

Never attempt to:

- Gain unauthorized access
- Escalate privileges
- Disrupt services
- Damage systems
- Exploit vulnerabilities without permission

Use intentionally vulnerable training applications only when they are specifically designed for security education.

---

### ✅ Separate Learning from Client Work

Learning exercises should remain separate from production projects.

Demo environments allow experimentation without introducing unnecessary risk to business systems or customer data.

---

### ✅ Practice Responsible Disclosure

If a legitimate security issue is discovered on an authorized platform:

- Document the findings clearly
- Follow the organization’s disclosure process
- Report issues responsibly
- Never publicly disclose vulnerabilities without authorization
